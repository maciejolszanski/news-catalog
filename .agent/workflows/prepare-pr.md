// turbo-all
1. Do it only if I ask you explicitly for preparing a PR
2. Run tests with `uv run pytest`
3. Verify that you are currently on a feature branch (not `main` or `master`).
4. Update the `README.md` with recent changes, installation steps, or usage instructions.
5. Stage all changes using `git add .`.
6. Commit changes with a descriptive message following conventional commits (e.g., `docs: update readme with simulator instructions`).
7. Push the changes to the remote repository.
